LLM USED? ✘ No

Stage 4 · Consolidation

Stage 5 · Validation
LLM USED? ✔ ValidationCrew
MODEL: DeepSeek Chat
1 Agent: Ambiguity, conflict & quality validation
Stage 6 · Publisher
LLM USED? ✘ No
CREW: None
PURPOSE: Artifact generation, serialization & caching
Overall LLM Strategy
  • ✔ 3 CrewAI crews with @CrewBase + YAML configs
  • ✔ Retry/backoff on all LLM calls (429/500/503)
  • ✔ Rules & deterministic for structure & parsing
  • ✔ 4 cache files for fast pipeline restarts
  • ✔ All DeepSeek (no Gemini quota issues)

📌 NOTES
• v2.0: 6-stage pipeline with 3 @CrewBase crews + 3 deterministic stages
• All LLM calls use DeepSeek (Chat + Reasoner) — no Gemini quota issues
• 4 cache files (extraction, consolidation, validation, quality) — fast restarts
• Exponential backoff retry on 429/500/503 for all LLM calls
• NEW Stage 4: Cross-issue consolidation with DeepSeek Reasoner for deep reasoning
• Proper CrewAI agents replace raw llm.call() — tool use, memory, structured output
